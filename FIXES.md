# Исправления путей в скриптах сборки

## Проблема
После перемещения скриптов в папку `scripts/` и конфигурационных файлов в `config/`, скрипты не могли найти нужные файлы.

## Исправления

### 1. `scripts/build.py`
- Теперь определяет корневую директорию проекта (на уровень выше `scripts/`)
- Переходит в корень проекта перед выполнением
- Все пути к файлам (`app.py`, `logo/`, `templates_example.json` и т.д.) теперь корректны

### 2. `scripts/build_exe.bat`
- Добавлен переход в корневую директорию: `cd /d "%~dp0\.."`
- Путь к скрипту изменен на: `python scripts\build.py`
- Путь к результату сборки: `dist\UTMka.exe` (относительно корня)

### 3. `scripts/build_installer.bat`
- Добавлен переход в корневую директорию: `cd /d "%~dp0\.."`
- Путь к setup.iss изменен на: `config\setup.iss`
- Добавлена проверка наличия Inno Setup

### 4. `scripts/build_app.bat`
- Добавлен переход в корневую директорию: `cd /d "%~dp0\.."`
- Путь к скрипту изменен на: `python scripts\build.py`

### 5. `config/setup.iss`
- Файл остается без изменений, так как Inno Setup запускается из корня проекта
- Все пути в setup.iss (`logo\logoutm.ico`, `dist\UTMka.exe`) корректны относительно корня

## Использование

Теперь все скрипты можно запускать из папки `scripts/`:

```bash
# Сборка exe файла
scripts\build_exe.bat

# Сборка установщика (после сборки exe)
scripts\build_installer.bat
```

Или двойным кликом по файлам в папке `scripts/`.
